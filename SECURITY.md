# 安全性说明

## 敏感信息保护

本项目作为开源工具，**不会提交或存储任何用户敏感信息**。所有敏感数据均由用户在本地环境配置，不会被上传到任何服务器。

### 需要用户自行配置的文件

以下文件**不会**包含在仓库中，需要用户在本地创建：

- `config.json` - 包含校园网账号、密码、网关地址等配置信息
  - 使用 `config.json.template` 作为模板创建
  - **不要将 `config.json` 提交到 Git 仓库**

### 自动生成的文件

以下文件会在运行时自动生成，已加入 `.gitignore`：

- `campus_connect.log` - 运行日志
- `*.png` - 错误截图文件
- `run_hidden.vbs` - 静默运行脚本（包含绝对路径）

## 安全最佳实践

1. **不要提交配置文件**
   ```bash
   # 确保 .gitignore 中包含
   config.json
   ```

2. **检查已提交的敏感信息**
   ```bash
   # 如果不小心提交了 config.json，立即删除
   git rm --cached config.json
   git commit -m "Remove sensitive config file"
   ```

3. **使用环境变量（可选）**
   如果需要更高的安全性，可以考虑修改代码使用环境变量存储敏感信息：
   ```bash
   # 设置环境变量
   export CAMPUS_USERNAME="your_username"
   export CAMPUS_PASSWORD="your_password"
   ```

4. **定期更换密码**
   - 建议定期更换校园网密码
   - 如果账号密码泄露，立即联系校园网服务中心

5. **权限管理**
   ```bash
   # 设置配置文件权限仅限当前用户
   chmod 600 config.json
   ```

## 账号密码存储方式

本项目默认使用 `config.json` 明文存储账号密码，原因如下：

- **校园网认证系统不支持加密存储**
- 账号密码只在本地使用，不会上传到任何服务器
- 用户完全控制自己的配置文件

如果您需要更高的安全性，请：
1. 将 `config.json` 添加到 `.gitignore`
2. 不要将配置文件分享给他人
3. 定期更换密码

## 漏洞报告

如果您发现安全漏洞，请不要公开提交 Issue。请通过以下方式私密报告：

1. GitHub Security Advisory（推荐）
2. 发送邮件至项目维护者

## 网络安全

- 本工具**仅**用于连接校园网认证网关
- 不会捕获、存储或传输任何网络流量
- 不会修改系统网络配置
- 不会安装任何网络监听或代理程序
